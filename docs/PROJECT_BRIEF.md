# Indica Project Brief

Indica is a living almanac for India: a public website that gathers scattered public data, turns it into sourced charts, and explains what the numbers mean in clear language. The product starts with India and is built so the same machinery can later support other countries.

The core problem is not a lack of statistics. India has too many statistics in too many places: RBI portals, MoSPI tables, ministry PDFs, World Bank series, survey microdata, climate APIs, air-quality feeds, and international datasets. Normal readers cannot discover, compare, or interpret them easily. Indica exists to make that reality legible.

## Product Promise

Every important public-interest question about India should eventually have a durable page:

- a precise question as the title;
- one primary chart;
- one or more supporting charts;
- a plain-language bottom line;
- layered explanations for different readers;
- source, method, caveat, and download links;
- stable URLs that can be cited and shared.

Indica is not a dashboard and not a news feed. It is an evergreen reference site with editorial judgment. Data stories can come later, but the compounding asset is the chart/question catalog.

## Editorial Model

The unit of publishing is a question, not a dataset.

Examples:

- How big is India's economy?
- How many people live in India?
- How long do Indians live?
- How bad is India's air pollution?
- How hot are Indian cities becoming?
- Why does GDP growth not feel like salary growth?
- Which states are pulling away from the rest?

Datasets are ingredients. The catalog decides which questions matter and which measure answers them best.

The operating editorial standard lives in [EDITORIAL_NORTH_STAR.md](./EDITORIAL_NORTH_STAR.md). In short: a question is only publishable when the page also has an answer thesis, a locked evidence packet, a chart list with distinct narrative jobs, and a visible caveat about what the data cannot prove.

## Explanation Model

Each chart page has one always-visible bottom line and three explanation registers:

- `reading`: analyst voice, precise terms, suitable for citations.
- `plain`: zero-jargon explanation for a general reader.
- `ground`: warm, grounded India-first framing with lived analogies.

Numbers are never generated by a language model. The data pipeline calculates and locks the numbers first. The model only writes prose around a structured evidence packet. Explanations are generated at ingest time, cached as content, and reviewed for important pages.

## MVP Scope

The first version should prove the full system on a narrow but credible catalog:

- Economy: GDP, growth, inflation, GDP per person, fiscal indicators.
- People: population, fertility, age structure, urbanization.
- Health: life expectancy, infant mortality, nutrition, NFHS indicators.
- Climate and air: PM2.5, AQI, heat, rainfall, dangerous nights.
- Energy: electricity access, generation mix, renewables, power demand.

Launch with 10-25 high-quality pages instead of hundreds of thin pages. The system should be able to hold thousands later.

## Non-Negotiables

- No third-party API call at request time.
- Static-first pages with inline SVG for primary charts.
- Source provenance attached to every published number.
- Raw source snapshots preserved separately from normalized data.
- Transformations are explicit, versioned, and reproducible.
- LLM prose is content, not runtime behavior.
- Multilingual data model from day one, even if the first launch is English-only.
